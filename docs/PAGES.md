# FastCloud — карта страниц (с приоритетами для MVP)

VPS-сервис · Европа · позиционирование «самые быстрые серверы»
Аналог по UX-простоте: DigitalOcean. Аудитория: DevOps · sysadmins · агентства · VPN-юзеры.

Список собран на основе анализа конкурентов: DigitalOcean, Hetzner, Vultr, Linode, Timeweb, Selectel, Beget.

---

## 🎯 Цель: запустить как можно быстрее

Используются три тира:

- ✅ **MVP** — без этого запускаться нельзя. Делается в первую очередь.
- 🟡 **v1.1** — нужно быстро добавить после запуска, первый старт переживёт.
- ⬜ **Позже** — можно отложить надолго или не делать совсем.

**Итоговый MVP-стек: ≈ 13 экранов.** Остальное — в очередь.

---

## 🌐 Маркетинговый сайт

### Основные
1. ✅ **Главная** — герой, преимущества, тарифы-превью, CTA. Без неё непонятно, что вы продаёте.
2. ✅ **VPS / Тарифы** — конфигуратор и таблица планов. Главная транзакционная страница.
3. 🟡 **Возможности / Почему мы** — на старте достаточно секции на главной; отдельную страницу можно позже.
4. 🟡 **Локации / Дата-центры** — на старте — секция на главной/тарифах. Отдельная страница нужна для SEO.
5. ⬜ **ОС и образы** — выбор ОС видно в конфигураторе. Отдельная страница не критична.

### Use-case (для SEO)
6. ⬜ VPS для разработки
7. ⬜ VPS для игровых серверов
8. ⬜ VPS для веб-хостинга / 1С-Битрикс
9. ⬜ VPS для VPN / прокси
10. ⬜ Выделенные серверы

> SEO-страницы — это рост во второй половине года, не блокер запуска.

### Поддержка и контент
11. 🟡 **База знаний / Документация** — минимум 5–10 статей нужно, иначе поддержка утонет. Можно стартовать с FAQ-странички.
12. ⬜ **Блог** — контент-маркетинг, не блокер.
13. ⬜ **Status page** — на старте можно использовать готовый сервис (Statuspage, Better Uptime).
14. ✅ **Контакты / Поддержка** — минимум email + Telegram. На старте этого достаточно вместо тикетов.
15. ⬜ **О компании** — желательно, но не блокер запуска.

### Юридические и служебные
16. ✅ **Договор-оферта, Политика конфиденциальности** — без них нельзя принимать платежи в EU. **SLA можно черновиком.**
17. ⬜ **Партнёрская / реферальная программа** — драйвер роста, но не блокер.
18. ✅ **Вход / Регистрация / Восстановление пароля** — обязательно.

## 🖥 Личный кабинет — серверы

### Главное
1. ✅ **Дашборд** — баланс, активные серверы, кратко статус
2. ✅ **Мои серверы** — список с фильтрами и быстрыми действиями
3. ✅ **Карточка сервера** — вкладки:
   - ✅ Обзор
   - ✅ Консоль (VNC / SSH в браузере) — критично, главная фишка
   - ✅ Питание (старт/стоп/перезагрузка)
   - 🟡 Графики нагрузки — на старте можно ограничиться текущими значениями
   - ✅ Сеть (показ IP, обратный DNS)
   - 🟡 Бэкапы — клиенты ждут, но v1 переживёт без авто-бэкапов
   - 🟡 Снапшоты — то же самое
   - ✅ Перестановка ОС
   - ✅ Ресайз тарифа
   - 🟡 Файрвол — на старте можно отдать в iptables на сервере
4. ✅ **Создание нового VPS** — пошаговый или one-page конфигуратор

### Инфраструктура
5. 🟡 Снапшоты и бэкапы (отдельная страница списка)
6. 🟡 Сеть · IP · обратный DNS · приватные сети — отдельная страница не критична, всё внутри карточки
7. ✅ **SSH-ключи** — для VPS это must-have
8. 🟡 Файрвол / профили правил
9. ⬜ DNS-зоны
10. ⬜ Объектное хранилище / S3

## 💳 Личный кабинет — финансы

16. 🟡 **Автоплатёж**

## ✅ Финальный MVP-чеклист (≈ 13 экранов)

### Публичная часть (5)
- [ ] Главная
- [ ] Тарифы (с конфигуратором)
- [ ] Контакты
- [ ] Юридические страницы (оферта, политика, SLA)
- [ ] Вход / Регистрация / Восстановление пароля

### Кабинет (8)
- [ ] Дашборд
- [ ] Список серверов
- [ ] Создание VPS
- [ ] Карточка сервера + веб-консоль
- [ ] SSH-ключи
- [ ] Биллинг (баланс + операции + пополнение + способы оплаты)
- [ ] Профиль

### За скобками MVP, но обсудить отдельно
- 2FA — security-must, выкатить в первый месяц
- Бэкапы / снапшоты — клиентское ожидание, второй спринт
- API + ключи — для DevOps-аудитории второй спринт
- База знаний — параллельно с запуском, минимум 10 статей
- Status page — взять готовый сервис
